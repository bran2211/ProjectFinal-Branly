Descripción

Este proyecto tiene como objetivo simular y configurar una red en Cisco Packet Tracer que incorpore los servicios esenciales de red, como DNS, DHCP, HTTP y HTTPS. La simulación permitirá observar cómo funcionan estos protocolos en una red, su configuración, interacción y cómo cada uno contribuye al funcionamiento eficiente de la comunicación en redes. El proyecto incluye tanto la configuración de los dispositivos de red como la implementación de los servidores y servicios para simular un entorno de red real.

Características Principales:

Servidor DNS:
Se configura un servidor DNS (Domain Name System) encargado de traducir nombres de dominio a direcciones IP y viceversa. Esto facilita la navegación por la red, ya que los usuarios pueden acceder a sitios web mediante nombres fáciles de recordar, en lugar de direcciones IP numéricas.

Características clave:
-Resolución de nombres de dominio a IP.
-Mantenimiento de registros DNS como A, MX, CNAME.

Servidor DHCP:
El servidor DHCP (Dynamic Host Configuration Protocol) asigna automáticamente direcciones IP y otros parámetros de red (como la puerta de enlace predeterminada y servidores DNS) a los dispositivos que se conectan a la red, evitando la necesidad de configurarlos manualmente.

Características clave:
-Asignación automática de direcciones IP.
-Configuración de opciones como máscara de subred y gateway.
-Simplificación de la administración de red en redes dinámicas.

Servidor HTTP:
El servidor HTTP (HyperText Transfer Protocol) permite la visualización de sitios web en la red local mediante el protocolo de transferencia de hipertexto. Los usuarios pueden acceder a contenido web desde navegadores utilizando el protocolo HTTP.

Características clave:
-Transferencia de datos en forma de páginas web.
-Uso del puerto 80 para comunicación.
-Ideal para simular sitios web internos en redes locales.

Servidor HTTPS:
El servidor HTTPS (HyperText Transfer Protocol Secure) extiende el protocolo HTTP añadiendo una capa de seguridad mediante el uso de SSL/TLS (Secure Sockets Layer/Transport Layer Security), proporcionando encriptación para garantizar la integridad y confidencialidad de los datos transmitidos.

Características clave:
- Comunicación segura y encriptada.
-Uso del puerto 443 para comunicación.
-Autenticación del servidor para proteger a los usuarios contra ataques de intermediarios.
